# 功能需求文档

## 一、项目基本信息

**项目名称**：课程作业提交管理小工具  
**目标用户**：大学生  
**技术栈**：React + localStorage  
**开发方式**：前端单页应用，无需后端服务

---

## 二、功能需求列表

### 2.1 基础功能（必须实现）

#### FR-1: 添加课程作业
**功能描述**：用户可以创建新的作业记录  
**输入信息**：
- 课程名称（必填）
- 作业标题（必填）
- 截止日期（必填，日期选择器）
- 完成状态（默认：未完成）

**业务规则**：
- 课程名和作业标题不能为空
- 截止日期不能早于当前日期（可选校验）
- 添加后自动保存到localStorage

**验收标准**：
- ✅ 填写完整信息后可成功添加
- ✅ 必填字段有验证提示
- ✅ 添加后立即显示在列表中

---

#### FR-2: 编辑作业信息
**功能描述**：用户可以修改已有作业的各项信息  
**可编辑字段**：课程名、作业标题、截止日期、完成状态

**业务规则**：
- 编辑后自动保存
- 取消编辑恢复原数据

**验收标准**：
- ✅ 点击编辑进入编辑模式
- ✅ 修改后保存成功
- ✅ 取消编辑数据不变

---

#### FR-3: 删除作业
**功能描述**：用户可以移除不需要的作业记录  
**交互方式**：删除按钮 + 确认提示

**业务规则**：
- 删除前需二次确认
- 删除后从列表和存储中移除

**验收标准**：
- ✅ 点击删除弹出确认框
- ✅ 确认后作业从列表消失
- ✅ 取消删除作业保留

---

#### FR-4: 标记完成状态
**功能描述**：用户可以标记作业为已完成或未完成  
**状态类型**：已完成 / 未完成

**业务规则**：
- 点击切换状态
- 已完成项视觉上与未完成项有区分

**验收标准**：
- ✅ 点击状态按钮立即切换
- ✅ 已完成项显示完成标记
- ✅ 状态保存到localStorage

---

#### FR-5: 按课程筛选
**功能描述**：用户可以选择查看特定课程的作业  
**筛选方式**：下拉选择课程名

**业务规则**：
- 支持选择"全部课程"
- 筛选结果实时更新

**验收标准**：
- ✅ 选择课程后仅显示该课程作业
- ✅ 选择"全部"显示所有作业
- ✅ 筛选器显示所有课程选项

---

#### FR-6: 按完成状态筛选
**功能描述**：用户可以筛选已完成或未完成的作业  
**筛选选项**：全部 / 已完成 / 未完成

**业务规则**：
- 默认显示"全部"
- 与课程筛选可组合使用

**验收标准**：
- ✅ 筛选结果准确
- ✅ 与课程筛选同时生效
- ✅ 切换筛选立即更新列表

---

#### FR-7: 数据持久化
**功能描述**：页面刷新或关闭后，数据仍然保留  
**存储方式**：localStorage

**数据结构**：
```javascript
{
  id: string,          // 唯一标识
  courseName: string,  // 课程名
  title: string,       // 作业标题
  deadline: string,    // 截止日期 (YYYY-MM-DD)
  completed: boolean   // 完成状态
}
```

**业务规则**：
- 每次操作自动保存
- 页面加载自动恢复
- 支持至少100条记录

**验收标准**：
- ✅ 刷新页面数据完整保留
- ✅ 关闭浏览器重开数据仍在
- ✅ 数据格式正确无误

---

### 2.2 扩展功能（已选择）

#### EX-1: 按截止日期排序 ⭐
**功能描述**：用户可以按截止日期对作业列表排序  
**排序方式**：升序（最近在前）/ 降序（最远在前）

**业务规则**：
- 默认按截止日期升序排列
- 提供排序切换按钮
- 逾期任务优先显示

**验收标准**：
- ✅ 支持升序/降序切换
- ✅ 默认最近截止日期在前
- ✅ 排序状态有视觉指示
- ✅ 空日期处理合理

---

#### EX-2: 逾期任务高亮显示 ⭐
**功能描述**：超过截止日期且未完成的作业用醒目方式标注  
**高亮方式**：红色背景/红色边框/红色标记

**业务规则**：
- 当前日期 > 截止日期 且 未完成 → 高亮
- 已完成的逾期任务不高亮

**验收标准**：
- ✅ 逾期未完成任务有醒目标记
- ✅ 已完成任务不高亮
- ✅ 未逾期任务正常显示
- ✅ 高亮颜色醒目但不刺眼

## 三、界面需求

```
┌─────────────────────────────────────┐
│  页面标题                            │
├─────────────────────────────────────┤
│  筛选栏：课程筛选 | 状态筛选 | 排序   │

```

├── Header (页面标题)

- **空状态**：显示"暂无作业，点击添加"

## 四、数据需求

### 4.1 数据模型
```javascript
// 作业对象
Assignment {
  id: string,          // UUID
  courseName: string,  // 课程名
  title: string,       // 作业标题
  deadline: string,    // 截止日期 ISO格式
  completed: boolean,  // 是否完成
  createdAt: string,   // 创建时间
  updatedAt: string    // 更新时间
}

// 应用状态
AppState {
  assignments: Assignment[],  // 作业列表
  filters: {
    course: string,           // 课程筛选
    status: 'all' | 'completed' | 'pending',  // 状态筛选
    sortBy: 'deadline',       // 排序字段
    sortOrder: 'asc' | 'desc' // 排序方向
  }
}
```

### 4.2 存储方案
- **存储位置**：localStorage
- **存储键名**：`assignment_manager_data`
- **存储时机**：每次数据变更立即保存
- **读取时机**：应用初始化时读取

### 4.3 数据验证
- 课程名：非空字符串，最大长度50字符
- 作业标题：非空字符串，最大长度100字符
- 截止日期：有效的日期格式
- 完成状态：布尔值

## 五、技术实现要求

### 5.1 开发环境
- Node.js >= 14.x
- npm 或 yarn
- 现代浏览器（支持ES6+）

### 5.2 技术选型
| 类别 | 技术选择 | 说明 |
|------|----------|------|
| 框架 | React 18 | 使用函数组件 + Hooks |
| 状态管理 | useState + useEffect | 不引入Redux |
| 路由 | 无需路由 | 单页面应用 |
| 样式 | CSS Modules | 避免样式冲突 |
| 构建工具 | Create React App | 快速搭建 |

### 5.3 关键技术点
- **localStorage操作**：封装统一的存储读写函数
- **日期处理**：使用Date对象或轻量级库
- **筛选逻辑**：数组filter + sort链式调用
- **响应式更新**：React状态更新触发重渲染

## 九、开发约束

### 9.1 不做的事
- ❌ 不实现用户登录注册
- ❌ 不实现后端服务
- ❌ 不实现数据云同步
- ❌ 不实现复杂动画效果
- ❌ 不实现多语言支持

### 9.2 代码规范
- 使用函数组件和Hooks
- 组件命名使用PascalCase
- 函数命名使用camelCase
- 关键逻辑添加注释
- 避免过长的函数和组件

### 9.3 性能要求
- 列表渲染使用key属性
- 避免不必要的状态更新
- localStorage操作添加try-catch
