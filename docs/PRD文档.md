# 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品愿景
为大学生提供一个简洁、高效的课程作业管理工具，帮助学生更好地规划和跟踪学习任务，避免遗漏重要作业和错过截止日期。

### 1.2 产品目标
- **核心目标**：帮助学生有效管理课程作业，提升学习规划能力
- **用户价值**：
  - 集中管理所有课程的作业信息
  - 直观查看作业完成进度
  - 及时发现逾期任务
  - 数据本地持久化，无需登录注册

### 1.3 产品定位
- **类型**：轻量级前端单页应用
- **用户群**：大学生群体
- **使用场景**：个人学习管理、作业进度跟踪
- **部署方式**：本地运行，数据存储在浏览器

---

## 二、用户画像

### 2.1 主要用户：大学生

**基本信息**：
- 年龄：18-25岁
- 特征：同时学习多门课程，每门课程都有作业任务
- 痛点：容易忘记作业截止日期，难以统筹安排学习计划

**典型场景**：
1. 一学期有5-8门课程，每门课程都有多个作业
2. 作业类型多样：论文、实验报告、编程作业、小组作业等
3. 截止日期集中在某些时段，容易遗漏
4. 希望能快速查看还有哪些作业未完成

**期望**：
- 一目了然地看到所有待完成作业
- 快速添加、编辑和删除作业记录
- 能够按课程或状态筛选查看
- 数据不会因为关闭浏览器而丢失

---

## 三、用户故事与场景

### US-1: 添加新作业
**作为**一个学生  
**我想要**快速记录新布置的作业信息  
**以便**不会忘记作业的存在和截止日期

**验收标准**：
- 可以填写课程名、作业标题、截止日期
- 添加后立即显示在列表中
- 数据自动保存

---

### US-2: 查看待完成作业
**作为**一个学生  
**我想要**筛选查看所有未完成的作业  
**以便**集中精力处理当前任务

**验收标准**：
- 可以选择只看未完成的作业
- 筛选结果准确
- 可以切换回查看全部

---

### US-3: 标记作业完成
**作为**一个学生  
**我想要**在完成作业后标记其为已完成  
**以便**区分已完成和未完成的任务

**验收标准**：
- 可以一键切换完成状态
- 已完成任务有视觉区分
- 状态保存到本地存储

---

### US-4: 删除过期作业
**作为**一个学生  
**我想要**删除不再需要的作业记录  
**以便**保持列表的整洁

**验收标准**：
- 可以删除单条作业
- 删除前有确认提示
- 删除后列表立即更新

---

### US-5: 按课程查看作业
**作为**一个学生  
**我想要**只看某门课程的作业  
**以便**集中处理特定课程的任务

**验收标准**：
- 可以选择特定课程
- 显示该课程的所有作业
- 可以切换回查看全部课程

---

### US-6: 发现逾期作业
**作为**一个学生  
**我想要**一眼看出哪些作业已经逾期  
**以便**优先处理紧急任务或与老师沟通

**验收标准**：
- 逾期且未完成的作业有明显标记
- 已完成的逾期作业不高亮
- 高亮效果醒目但不刺眼

---

### US-7: 按截止日期排序
**作为**一个学生  
**我想要**按截止日期排序查看作业  
**以便**知道哪些作业最紧急

**验收标准**：
- 可以按升序或降序排列
- 默认最近截止日期在前
- 排序状态有视觉指示

---

## 五、非功能需求

### 5.1 性能需求
- **响应时间**：操作响应 < 100ms
- **数据容量**：支持至少100条作业记录
- **页面加载**：首屏加载 < 1秒（本地运行）

### 5.2 兼容性需求
- **浏览器支持**：
  - Chrome (推荐)
  - Edge
  - Firefox
  - Safari
- **屏幕适配**：支持主流笔记本和台式机分辨率

### 5.3 可用性需求
- **学习成本**：无需说明书，界面直观易懂
- **操作流程**：添加作业 < 3次点击
- **视觉设计**：简洁清晰，信息层次分明

### 5.4 可维护性需求
- **代码结构**：组件化开发，职责清晰
- **注释覆盖**：关键逻辑有注释说明
- **文档完备**：有README和使用说明

## 六、产品边界与约束

### 6.1 功能边界（做什么）
✅ **核心功能**
- 作业的增删改查
- 筛选和排序
- 本地数据持久化
- 逾期任务提醒

✅ **用户体验**
- 简洁直观的界面
- 即时的操作反馈
- 醒目的逾期提示

### 6.2 功能边界（不做什么）
❌ **超出范围**
- 用户注册登录系统
- 后端服务和数据库
- 多人协作和分享
- 数据云同步
- 作业内容编辑器
- 文件上传附件
- 邮件/短信提醒
- 移动端适配
- 数据导出功能

### 6.3 技术约束
- **前端框架**：React (函数组件 + Hooks)
- **状态管理**：React内置状态管理
- **样式方案**：CSS Modules
- **构建工具**：Create React App
- **数据存储**：localStorage (约5MB限制)

### 6.4 开发约束
- **开发时间**：实验课时间范围内
- **团队规模**：小组协作
- **技术难度**：适合初学者，避免过度设计
- **代码质量**：可读性好，结构清晰

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| localStorage空间不足 | 数据丢失 | 控制数据量，添加错误处理 |
| 浏览器隐私模式 | 数据不持久 | 提示用户使用正常模式 |
| 日期格式不一致 | 逾期判断错误 | 统一使用ISO日期格式 |
| React学习曲线 | 开发效率低 | 参考官方文档和示例 |

## 十、术语表

| 术语 | 定义 |
|------|------|
| 作业 (Assignment) | 一项需要完成的学习任务，包含课程名、标题、截止日期等信息 |
| 逾期 (Overdue) | 当前日期超过截止日期且未完成的状态 |
| localStorage | 浏览器提供的本地存储机制，数据持久化保存在用户设备上 |
| React组件 | React应用的基本构建单元，可复用的UI模块 |
| 筛选 (Filter) | 根据特定条件过滤显示的数据项 |
| 排序 (Sort) | 按照特定规则排列数据项的顺序 |
